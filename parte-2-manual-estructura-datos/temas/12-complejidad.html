Tema 12

Complejidad algorítmica

La complejidad algorítmica es una medida que indica cuánto tiempo y/o espacio requiere un algoritmo para ejecutarse en función del tamaño de la entrada (n).

Documentación

Definición de complejidad algorítmica

La complejidad algorítmica es una medida que indica cuánto tiempo y/o espacio requiere un algoritmo para ejecutarse en función del tamaño de la entrada (n).

Tipos de complejidad

  • Complejidad temporal: Mide el tiempo que tarda un algoritmo en ejecutarse.
  • Complejidad espacial: Mide la cantidad de memoria que utiliza.

Notación Big-O (O grande)

Se utiliza para expresar el comportamiento del algoritmo en el peor de los casos.

Ejemplos comunes:

  • O(1) — Constante
  • O(log n) → Logarítmica
  • O(n) → Lineal
  • O(n log n) → Lineal logarítmica
  • O(n^2) → Cuadrática

Orden de crecimiento (de mejor a peor):

O(1) < O(log n) < O(n) < O(n log n) < O(n^2)

Casos de análisis

  • Mejor caso: ejecución más rápida posible
  • Caso promedio: comportamiento habitual
  • Peor caso: ejecución más lenta

Características

  • No mide tiempo real (segundos).
  • Depende del tamaño de entrada.
  • Permite comparar algoritmos.
  • Ignora constantes y términos menores.

Ventajas

  • Permite elegir algoritmos eficientes.
  • Ayuda a optimizar código.
  • Es independiente del hardware.

Desventajas

  • No muestra tiempo exacto.
  • Puede ser abstracto.
  • Requiere análisis matemático.

Usos comunes

  • Optimización de programas.
  • Análisis de rendimiento.
  • Comparación de algoritmos.
  • Diseño de software eficiente.

Ejemplos

Ejemplo 1: Complejidad constante O(1)

Un sistema accede directamente a un elemento de un arreglo sin importar su tamaño.

Pseudocódigo explicativo

INICIO

1. Tener un arreglo. 

2. Acceder a una posición específica. 

3. Mostrar el valor. FIN
Java 

public class Main {
    public static void main(String[] args) {
        int[] arreglo = {10, 20, 30, 40, 50};
        System.out.println("Elemento en posición 2: " + arreglo[2]);
    }
}
Salida esperada 
Elemento en posición 2: 30

Ejemplo 2: Complejidad lineal O(n)

Un sistema recorre todos los elementos de un arreglo para mostrarlos.

Pseudocódigo explicativo

INICIO

1. Tener un arreglo. 

2. Recorrer todos los elementos. 

3. Mostrar cada uno. FIN
Java 

public class Main {
    public static void main(String[] args) {
        int[] arreglo = {10, 20, 30, 40, 50};

        for (int i = 0; i < arreglo.length; i++) {
            System.out.println(arreglo[i]);
        }
    }
}
Salida esperada 
10
20
30
40
50

Retos

Reto 1: Complejidad cuadrática O(n²)

Crear un programa que compare todos los elementos de un arreglo con todos los demás.

Pistas

  • Usar dos ciclos anidados.
  • Cada ciclo recorre n elementos.
  • n × n = n².
Salida esperada 
Comparaciones realizadas: 25

Reto 2: Identificar la complejidad

Analizar el siguiente código y determinar su complejidad:

for (int i = 0; i < n; i++) { for (int j = 0; j < n; j++) { System.out.println(i + " " + j); } }

Pistas

  • Hay dos ciclos.
  • Cada uno recorre n veces.
Salida esperada 
Complejidad: O(n²)
