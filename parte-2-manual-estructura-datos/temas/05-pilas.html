Tema 5

Pilas Stack

Implementación de estructuras tipo pila, operaciones LIFO y aplicaciones prácticas en programación.

Documentación

Definición de pila (Stack)

Una pila es una estructura de datos lineal que sigue el principio LIFO (Last In, First Out), es decir, el último elemento en entrar es el primero en salir. Se puede imaginar como una pila de platos, donde solo se puede agregar o quitar elementos desde la parte superior.

Operaciones básicas de una pila

  • Push (apilar): Agrega un elemento en la parte superior de la pila.
  • Pop (desapilar): Elimina el elemento que está en la parte superior.
  • Peek (cima): Muestra el elemento superior sin eliminarlo.
  • isEmpty (vacía): Verifica si la pila está vacía.

Características de la pila

  • Solo permite acceso al elemento superior.
  • Sigue el principio LIFO.
  • No permite acceso directo a elementos intermedios.
  • Puede implementarse con arreglos o listas enlazadas.

Ventajas de la pila

  • Estructura simple y fácil de implementar.
  • Útil para controlar procesos con orden inverso.
  • Eficiente en inserción y eliminación (O(1)).

Desventajas de la pila

  • Acceso limitado a un solo elemento (la cima).
  • No es adecuada cuando se necesita acceso aleatorio.

Usos comunes

  • Deshacer y rehacer acciones (Undo/Redo).
  • Evaluación de expresiones matemáticas.
  • Navegación hacia atrás en aplicaciones.
  • Llamadas a funciones (pila de ejecución).
  • Validación de paréntesis en código.

Ejemplo 1: Registro de acciones (Undo)

Descripción

Un sistema guarda las acciones realizadas por un usuario en una pila. Cada nueva acción se agrega a la cima, y cuando el usuario desea deshacer, se elimina la última acción realizada.

Pseudocódigo explicativo 

INICIO

1. Crear una pila vacía.
2. Agregar varias acciones usando push.
3. Mostrar la acción en la cima.
4. Eliminar la última acción con pop.
5. Mostrar la nueva acción en la cima.

FIN.
Java 

import java.util.Stack;

public class Main {

    public static void main(String args[]) {

        Stack acciones = new Stack<>();

        acciones.push("Abrir archivo");
        acciones.push("Editar texto");
        acciones.push("Guardar cambios");
        acciones.push("Cerrar archivo");

        System.out.println(
            "===== HISTORIAL DE ACCIONES ====="
        );

        System.out.println(
            "Última acción: " + acciones.peek()
        );

        acciones.pop();

        System.out.println(
            "Se deshizo la última acción."
        );

        System.out.println(
            "Nueva última acción: " + acciones.peek()
        );

    }

}

Ejemplo 2: Validación de paréntesis

Descripción

Un programa necesita verificar si una expresión matemática tiene los paréntesis correctamente balanceados utilizando una pila.

Pseudocódigo explicativo 

INICIO

1. Crear una pila vacía.
2. Recorrer cada carácter de la expresión.
3. Si es "(", hacer push.
4. Si es ")", hacer pop.
5. Si en algún momento no hay qué desapilar, error.
6. Al final, si la pila está vacía, es válida.

FIN.
Java 

import java.util.Stack;

public class Main {

    public static boolean validarParentesis(
        String expresion
    ) {

        Stack pila = new Stack<>();

        for (int i = 0;
             i < expresion.length();
             i++) {

            char c = expresion.charAt(i);

            if (c == '(') {

                pila.push(c);

            } else if (c == ')') {

                if (pila.isEmpty()) {

                    return false;

                }

                pila.pop();

            }

        }

        return pila.isEmpty();

    }

    public static void main(String[] args) {

        String expresion = "(a+b)*(c+d)";

        System.out.println(
            "===== VALIDACIÓN DE PARENTESIS ====="
        );

        System.out.println(
            "Expresión: " + expresion
        );

        if (validarParentesis(expresion)) {

            System.out.println(
                "La expresión es válida."
            );

        } else {

            System.out.println(
                "La expresión es incorrecta."
            );

        }

    }

}

Retos

Reto 1: Historial de navegación (solo retroceso)

Un navegador almacena las páginas visitadas en una pila. El usuario solo puede regresar a la página anterior (no avanzar).

Pistas

  • Usa una pila para guardar las páginas.
  • Cada nueva página se agrega con push.
  • Para volver atrás, usa pop.
  • La página actual es la que está en la cima.

Reto 2: Invertir una lista usando pila

Un sistema necesita invertir el orden de una lista de elementos utilizando una pila.

Pistas

  • Inserta todos los elementos en la pila.
  • Luego, desapílalos para obtener el orden inverso.
  • La pila automáticamente invierte el orden por su comportamiento LIFO.
