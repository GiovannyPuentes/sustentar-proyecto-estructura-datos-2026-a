Tema 1

Introducción a las estructuras de datos

Fundamentos, clasificación, importancia y aplicaciones de las estructuras de datos en programación.

Documentación

Definición

Una estructura de datos es una forma de organizar, almacenar y gestionar información dentro de un programa para facilitar su acceso, modificación y procesamiento de manera eficiente.

Diferencia entre estructura de datos y algoritmo

Estructura de datos

Define cómo se almacenan y organizan los datos.

Algoritmo

Define los pasos que se siguen para trabajar con esos datos.

Ambos se complementan: la estructura organiza la información y el algoritmo la procesa.

Importancia

  • Permiten optimizar el rendimiento de los programas.
  • Facilitan el manejo de grandes cantidades de información.
  • Mejoran la claridad, orden y mantenimiento del código.
  • Ayudan a resolver problemas de forma más eficiente.

Características principales

Organización

Establece cómo se ordenan los datos.

Acceso

Define cómo se consultan los elementos.

Eficiencia

Influye en el tiempo y recursos usados.

Flexibilidad

Algunas estructuras permiten crecer o reducirse.

Relación entre datos

Puede ser lineal o no lineal.

Clasificación general

Estructuras lineales

Los elementos se organizan en secuencia.

Arrays, listas, pilas y colas.

Estructuras no lineales

Los elementos no siguen una sola secuencia.

Árboles y grafos.

Estructuras estáticas

Su tamaño no cambia fácilmente.

Estructuras dinámicas

Su tamaño puede variar durante la ejecución.

Usos comunes

  • Registro de estudiantes, usuarios o productos.
  • Gestión de turnos o solicitudes.
  • Navegación de historial.
  • Organización jerárquica de información.
  • Modelado de relaciones complejas como redes o rutas.

Ejemplo 1: Registro de estudiantes

Descripción

Un sistema básico necesita almacenar nombres de estudiantes, mostrar cuántos hay, identificar el primero y el último, y listar todos los registros.

Pseudocódigo explicativo

INICIO

1. Crear una lista de estudiantes.
2. Obtener la cantidad de estudiantes.
3. Mostrar la cantidad total.
4. Mostrar el primer estudiante.
5. Mostrar el último estudiante.
6. Recorrer la lista y mostrar todos los nombres.

FIN
Java 

public class Main {

    public static void main(String[] args) {

        // 1. Crear La Lista de estudiantes
        String[] estudiantes = {
            "Carlos",
            "María",
            "Juan",
            "Laura",
            "Andrés"
        };

        // 2. Obtener la cantidad de estudiantes
        int total = estudiantes.length;

        // 3. Mostrar información general
        System.out.println(
            "===== REGISTRO DE ESTUDIANTES ====="
        );

        System.out.println(
            "Total de estudiantes: " + total
        );

        // 4. Mostrar el primer y el último estudiante
        System.out.println(
            "Primer estudiante: " + estudiantes[0]
        );

        System.out.println(
            "Último estudiante: " +
            estudiantes[estudiantes.length - 1]
        );

        // 5. Recorrer la lista completa
        System.out.println("Listado completo:");

        for (int i = 0; i < estudiantes.length; i++) {

            System.out.println(
                (i + 1) + ". " + estudiantes[i]
            );

        }

    }

}

4. Laura

Ejemplo 2: Sistema de atención de clientes

Descripción

Un sistema de atención recibe clientes y debe atenderlos en orden de llegada, siguiendo la lógica FIFO: primero en entrar, primero en salir.

Pseudocódigo explicativo

INICIO

1. Crear una lista de clientes en espera.
2. Mostrar todos los clientes.
3. Identificar el primer cliente.
4. Atender al primer cliente.
5. Mostrar el cliente atendido.
6. Mostrar los clientes restantes.

FIN
Java 

public class Main {

    public static void main(String[] args) {

        // 1. Crear la cola de clientes
        String[] colaClientes = {
            "Cliente A",
            "Cliente B",
            "Cliente C",
            "Cliente D"
        };

        // 2. Mostrar el estado inicial
        System.out.println(
            "===== SISTEMA DE ATENCIÓN ====="
        );

        System.out.print(
            "Clientes en espera: "
        );

        for (int i = 0; i < colaClientes.length; i++) {

            System.out.print(
                colaClientes[i]
            );

            if (i < colaClientes.length - 1) {

                System.out.print(", ");

            }

        }

        System.out.println();

        // 3. Identificar el primer cliente
        String clienteActual = colaClientes[0];

        System.out.println(
            "Cliente en turno: " + clienteActual
        );

        // 4. Atender al cliente
        String clienteAtendido = clienteActual;

        System.out.println(
            "Cliente atendido: " + clienteAtendido
        );

        // 5. Mostrar los clientes restantes
        System.out.print(
            "Clientes restantes: "
        );

        for (int i = 1; i < colaClientes.length; i++) {

            System.out.print(
                colaClientes[i]
            );

            if (i < colaClientes.length - 1) {

                System.out.print(", ");

            }

        }

        System.out.println();

    }

}
Salida esperada 
===== SISTEMA DE ATENCIÓN =====

Clientes en espera:
Cliente A, Cliente B, Cliente C, Cliente D

Cliente en turno:
Cliente A

Cliente atendido:
Cliente A

Clientes restantes:
Cliente B, Cliente C, Cliente D

Retos

Reto 1: Catálogo de libros

Una biblioteca necesita almacenar una lista de libros y mostrar información básica: cantidad total, primer libro, último libro y listado completo.

Pistas

  • Usa un arreglo de tipo String.
  • La cantidad se obtiene con length.
  • El primer elemento está en la posición 0.
  • El último está en length - 1.
  • Recorre la lista con un ciclo for.
Salida esperada 
===== CATÁLOGO DE LIBROS =====

Total de libros: 5

Primer libro: El Quijote

Último libro: Rayuela

Listado:

1. El Quijote
2. Cien años de soledad
3. La Odisea
4. El Principito
5. Rayuela

Reto 2: Sistema de turnos

Un sistema de turnos recibe solicitudes numeradas. Debe mostrar la cola, atender el primer turno y luego mostrar los turnos restantes.

Pistas

  • Usa una estructura tipo cola.
  • El primer elemento es el primero en salir.
  • Muestra el estado antes y después de atender.
  • Recuerda separar correctamente los elementos al imprimirlos.

Turnos en espera:
T1, T2, T3, T4

Turno actual:
T1

Turno atendido:
T1

Turnos restantes:
T2, T3, T4
