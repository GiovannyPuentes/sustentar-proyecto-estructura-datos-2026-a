Tema 6

Colas Queue y variantes

Implementación de estructuras FIFO, colas circulares, colas de prioridad y aplicaciones prácticas en programación.

Documentación

Definición de cola (Queue)

Una cola es una estructura de datos lineal que sigue el principio FIFO (First In, First Out), es decir, el primer elemento en entrar es el primero en salir. Se puede comparar con una fila de personas, donde el primero en llegar es el primero en ser atendido.

Operaciones básicas de una cola

  • Enqueue (encolar): Agrega un elemento al final de la cola.
  • Dequeue (desencolar): Elimina el elemento que está al frente de la cola.
  • Front (frente): Muestra el primer elemento sin eliminarlo.
  • isEmpty (vacía): Verifica si la cola está vacía.

Características de la cola

  • Sigue el principio FIFO.
  • Inserción por un extremo (final) y eliminación por el otro (inicio).
  • No permite acceso directo a elementos intermedios.
  • Puede implementarse con arreglos o listas enlazadas.

Ventajas de la cola

  • Ideal para procesos ordenados por llegada.
  • Eficiente en inserción y eliminación (O(1)).
  • Fácil de implementar y entender.

Desventajas de la cola

  • Acceso limitado solo al primer elemento.
  • No permite recorrer libremente como otras estructuras.

Variantes de las colas

Cola circular

El último elemento se conecta con el primero.

Aprovecha mejor la memoria y evita espacios desperdiciados.

Cola de prioridad

Cada elemento tiene una prioridad.

Los elementos con mayor prioridad salen primero.

Cola doble (Deque)

Permite insertar y eliminar elementos por ambos extremos.

Usos comunes

  • Sistemas de atención al cliente (turnos).
  • Gestión de procesos en sistemas operativos.
  • Envío de documentos a una impresora.
  • Transmisión de datos en redes (buffers).

Ejemplo 1: Sistema de atención de clientes (Queue)

Descripción

Un banco recibe clientes y los atiende en estricto orden de llegada, siguiendo el principio FIFO.

Pseudocódigo explicativo 

INICIO

1. Crear una cola vacía.
2. Encolar varios clientes conforme llegan.
3. Mostrar el cliente que está al frente.
4. Desencolar al cliente atendido.
5. Mostrar el siguiente cliente en la fila.

FIN.
Java 

import java.util.LinkedList;
import java.util.Queue;

public class Main {

    public static void main(String args[]) {

        Queue clientes =
            new LinkedList<>();

        clientes.add("Cliente 1");
        clientes.add("Cliente 2");
        clientes.add("Cliente 3");
        clientes.add("Cliente 4");

        System.out.println(
            "===== SISTEMA DE ATENCIÓN ====="
        );

        System.out.println(
            "Siguiente en la fila: " +
            clientes.peek()
        );

        clientes.poll();

        System.out.println(
            "Cliente atendido."
        );

        System.out.println(
            "Nuevo siguiente en la fila: " +
            clientes.peek()
        );

    }

}

Ejemplo 2: Sistema de urgencias médicas (PriorityQueue)

Descripción

Un hospital recibe pacientes en urgencias. Los pacientes con mayor gravedad (prioridad más alta) deben ser atendidos primero, independientemente de su orden de llegada.

Pseudocódigo explicativo 

INICIO

1. Crear una cola de prioridad vacía.
2. Registrar pacientes asignándoles un nivel de prioridad.
3. Atender al paciente con mayor prioridad.
4. Eliminar al paciente atendido.
5. Mostrar el próximo paciente a atender.

FIN.
Java 

import java.util.PriorityQueue;

public class Main {

    public static void main(String args[]) {

        PriorityQueue pacientes =
            new PriorityQueue<>();

        pacientes.add(
            "Prioridad 3 - Paciente A"
        );

        pacientes.add(
            "Prioridad 1 - Paciente B"
        );

        pacientes.add(
            "Prioridad 2 - Paciente C"
        );

        System.out.println(
            "===== SISTEMA DE URGENCIAS ====="
        );

        System.out.println(
            "Paciente en atención: " +
            pacientes.peek()
        );

        pacientes.poll();

        System.out.println(
            "Paciente atendido."
        );

        System.out.println(
            "Siguiente paciente: " +
            pacientes.peek()
        );

    }

}

Reto 1: Gestión de impresión

Una impresora recibe documentos en una cola. Los documentos se imprimen en el orden en que fueron enviados.

Pistas

  • Usa una cola para almacenar documentos.
  • Cada documento se agrega al final.
  • El primero en la cola es el primero en imprimirse.
  • Simula la impresión eliminando elementos.
